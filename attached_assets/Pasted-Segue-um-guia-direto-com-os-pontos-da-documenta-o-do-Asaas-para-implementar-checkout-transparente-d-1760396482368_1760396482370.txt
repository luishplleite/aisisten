Segue um guia direto com os pontos da documentação do Asaas para implementar checkout transparente de assinatura e pagamentos avulsos, cobrindo cartão, dinheiro (boleto) e Pix, com rotas e exemplos de uso recomendados.[1]

### Visão geral
- O Asaas permite implementar checkout transparente via API, criando sessões de checkout conforme o fluxo desejado (avulso, assinatura, split), com suporte a cartão, boleto e Pix, sem redirecionar para páginas externas do provedor.[2][3][1]
- É possível informar previamente dados do cliente, configurar redirecionamentos de sucesso/cancelamento e controlar os métodos de pagamento disponíveis por checkout.[4][1]

### Checkout para assinatura (recorrente)
- A documentação de “Checkout com Assinatura (recorrente)” mostra como criar um checkout que, ao concluir com cartão, gera uma assinatura com cobranças automáticas futuras, mantendo a experiência transparente.[5]
- O fluxo típico: cadastrar/indicar o cliente, criar a sessão de checkout com tipo recorrente, habilitar cartão e definir URLs de redirecionamento; ao pagar, o Asaas cria a assinatura e programa as próximas cobranças.[5][4]

### Pagamentos avulsos (one‑time)
- Para avulsos, use o Asaas Checkout definindo produto/valor e métodos de pagamento (Pix, cartão, boleto), com validade do link e redirecionamentos; o pagamento gera uma cobrança única (payment) sem vínculo recorrente.[6][1][4]
- A página “Checkout para Pix” traz um exemplo completo de checkout avulso com Pix, incluindo tempo de expiração e nome do produto no link gerado.[6]

### Cartão de crédito (transparente)
- Há duas abordagens: via Checkout (gerando a sessão de checkout com opção de cartão) ou via API de pagamentos diretamente com “Pay a charge with credit card”/“Credit Card Charges” usando o endpoint de payments, mantendo o fluxo no próprio site.[7][8][1]
- A referência “Pay a charge with a credit card” descreve o endpoint para processar o pagamento de uma cobrança com dados do cartão no momento da chamada, sem agendamento, adequado para capturas imediatas.[7]

### Pix (QR dinâmico e checkout)
- A página “Checkout para Pix” mostra como criar a sessão de checkout já focada em Pix, com geração de QR Code/PIX copia e cola e expiração configurável por período curto (ex.: 1 hora).[6]
- O guia Pix consolida criação/uso de chaves, recebimento via QR dinâmico/estático e integrações do Pix na API para cobranças e pagamentos, servindo de base caso opte por construir sem a tela de Checkout pronta.[9][10]

### Boleto/Dinheiro (boleto bancário)
- A documentação de cobranças por boleto está na seção de “Bank Slip Charges” dentro de Payments; no Checkout, basta habilitar boleto como método para exibir linha digitável/baixar o PDF de boleto ao concluir.[8][1]
- Plugins oficiais e materiais confirmam compatibilidade de checkout transparente com cartão e boleto, refletindo as mesmas capacidades expostas na API.[11][12]

### Como informar dados do cliente e redirecionar
- É suportado preencher dados do cliente previamente para o Checkout (reduzindo fricção), ou deixar que o cliente preencha na tela; isso afeta UX, mas não muda a mecânica de criação/processamento.[1][4]
- “Link do checkout e redirecionamento do cliente” detalha como construir o link de sessão e definir URLs de retorno para sucesso/cancelamento, expirando a sessão quando apropriado.[4]

### Referências úteis por tópico
- Asaas Checkout: visão, criar sessão, opções de pagamento, dados do cliente, redirecionamentos.[3][1][4]
- Checkout com assinatura (recorrente): criação e comportamento ao pagar com cartão.[5]
- Checkout para Pix: exemplo de sessão com Pix, expiração e produto/valor.[6]
- Pagamento com cartão via API: endpoint “Pay a charge with credit card” e guia “Credit Card Charges”.[8][7]
- Pix na API: visão geral, QR dinâmico/estático, chaves e cobranças Pix.[10][9]
- Introdução/guia de cobranças: como usar a API de cobranças inclusive como checkout transparente por backend.[2]
